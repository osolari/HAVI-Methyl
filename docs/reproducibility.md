# Reproducibility

Every figure, table, and benchmark in the manuscript is regenerated by
a single script in `scripts/`. The canonical entry point is
`scripts/run_all.sh`, which drives the full pipeline. Outputs land in
`outputs/` and are kept **byte-identical** with the LaTeX-side mirrors
under `docs/report/` by the scripts themselves.

## CLI

```bash
bash scripts/run_all.sh                 # canonical pipeline + every CSV + figure
bash scripts/run_all.sh --fast          # smoke run (S=4, L=80, n_iter=3)
bash scripts/run_all.sh --figures       # only figures
bash scripts/run_all.sh --tables        # only tables
bash scripts/run_all.sh --benchmarks    # only benchmarks
make test                               # 156-test pytest suite
make report                             # regenerate figures + recompile docs/report/main.pdf
```

The script auto-detects Phase 5 assets (lab-drive availability, the
committed manifest, the high-variance panel) and falls back to the
synthetic FinaleMe-proxy when the Liu 2024 data is not mounted; see
the sticky-skip guard below.

## Canonical pipeline ordering

Order matters: `scripts/bench_synth_recovery.py` is the single source of
truth for `outputs/results.json` + `outputs/plot_data.npz`. Every Sec.
11 table and figure derives from those two artifacts. Running
`bench_synth_recovery.py` first guarantees that every downstream
artifact is internally consistent and that no figure script triggers an
expensive re-fit.

| Stage | Script | What it writes |
|---|---|---|
| 1. Canonical synthetic pipeline | `bench_synth_recovery.py` | `outputs/results.json`, `outputs/plot_data.npz`, `bench_synth_recovery.csv` |
| 2. Multi-seed CIs | `bench_multiseed_recovery.py` | `tab_recovery_multiseed.csv` |
| 3. Ablation matrix | `bench_ablation_matrix.py` | `bench_ablation_matrix.csv` (A0..A5) |
| 4. Torch SVI | `bench_torch_svi.py` | `bench_torch_svi.csv` (Gaussian vs flow, ELBO vs IWAE) |
| 5. Tissue LOO | `bench_tissue_loo.py` | `bench_tissue_loo.csv` |
| 6. Per-tissue breakdown | `fig_loyfer_loo_per_tissue.py` | `bench_loyfer_loo_per_tissue.csv` |
| 7. Liu 2024 real-data | `bench_finaleme_realdata.py` | `bench_finaleme_realdata.csv`, `bench_finaleme_coverage_strat.csv` |
| 8. Simulator validation | `bench_simulator_validation.py` | `bench_simulator_validation.csv` (App. H) |
| 9. Compute budget | `bench_compute_budget.py` | `bench_compute_budget.csv` (App. F) |
| 10. Figures | `fig_*.py` | PNG + PDF under `outputs/figures/` |
| 11. Tables | `tab_*.py` | CSV under `outputs/tables/` (LaTeX-side tables read these) |

## Sticky-skip guard

`scripts/bench_finaleme_realdata.py` ships a **sticky-skip guard**: if
`outputs/tables/bench_finaleme_realdata.csv` already contains a real
`_status="Liu 2024..."` row, the script will not overwrite it with the
synthetic-proxy fallback when the lab drive is unmounted. This is what
keeps the headline Pearson $r = 0.467$ row alive across local CPU
re-runs that cannot see `/Volumes/Omid Solari/finaleme/`.

To force a re-run, delete `bench_finaleme_realdata.csv` before invoking
`scripts/run_all.sh`, or pass `--force-realdata` to the bench script.

## `outputs/` ↔ `docs/report/` invariant

The mirror relationship between `outputs/` and `docs/report/` is
maintained as a **byte-identical** invariant by
`scripts/bench_synth_recovery.py` and the per-figure scripts. Every
PNG/PDF figure mirrored into `docs/report/figures/`, every CSV mirrored
into `docs/report/tables/`, the JSON mirrored into
`docs/report/results.json` and `docs/report/code/results.json` — all
written by the same Python process that produced the canonical
artifact, so the LaTeX build never diverges from the live pipeline.

If you change a number in the manuscript and the figure does not
update, the contract has been broken; re-run `bash scripts/run_all.sh`
and the invariant will heal.

## 156-test suite

```bash
make test                                   # full run
PYTHONPATH=src python -m pytest tests -v
```

Each test verifies a specific theoretical claim from the manuscript:

- Closed-form Gaussian-Gaussian KL formulas vs Monte Carlo
- Beta-Binomial log-pmf vs scipy reference
- Fano bound monotonicity in mutual information
- Hierarchical pooling shrinkage formula
- Conformal coverage at the nominal level (within $\pm 5\%$)
- Simulator validation against published cfDNA distributions
  (fragment-length primary mode, 320–350 bp peak, 10.4 bp periodicity,
  top-4 motif fraction, methylation-conditioned GC effect)
- Permutation invariance of the Set Transformer encoder
- Robbins-Monro recentering preserves $\sum_s \delta_s = 0$
- IWAE-DReG identity at $K=1$

9 are torch-conditional and skip on a numpy-only install.

## Compute environment notes

The headline Liu 2024 real-data row was produced by a $500$-iteration
training of `fit_svi_torch` on a single NVIDIA A10G GPU with mini-batch
sizes $|B_s|=16$, $|B_\ell|=128$ and $K=4$ IWAE samples. The numpy-only
synthetic benches run on CPU in minutes. The Phase 5 torch run uses
`a10-dev` over SSH (`ssh a10-dev`); the bench script auto-detects the
lab drive at `/cb/home/omids/data/finaleme` when run on that host, and
at `/Volumes/Omid Solari/finaleme/` when run locally.

!!! note "macOS Removable-Volumes TCC"
    On macOS the running VS Code process must have explicit
    Files-and-Folders access to *Removable Volumes* before the lab
    drive becomes visible. A fresh VS Code session may need to be
    relaunched from the Dock before
    `scripts/bench_finaleme_realdata.py` can see `/Volumes/Omid Solari/`.

## What's reproducible without network

- Every synthetic figure (`recovery_scatter`, `calibration`,
  `multiseed_recovery`, `elbo_trajectory` against the snapshot
  trajectory) and every synthetic table (`tab_recovery`,
  `tab_recovery_multiseed`, `bench_ablation_matrix`, `bench_torch_svi`,
  `bench_simulator_metrics`, `bench_compute_budget`).
- The sticky-skipped Liu 2024 row in `bench_finaleme_realdata.csv` and
  the matching `finaleme_paired_metrics`, `finaleme_paired_scatter`,
  `finaleme_coverage_strat` figures, provided the CSV is already
  populated from a previous lab-drive run.

## What needs the lab drive

- A fresh `bench_finaleme_realdata.csv` run on `/Volumes/Omid Solari/finaleme/`.
- A fresh `bench_tissue_loo.csv` run against the real Loyfer panel at
  `data/loyfer_panel/Atlas.U25.l4.hg38.tsv`.
- `make report`, which requires a pdfLaTeX installation
  (`brew install --cask basictex` is the smallest path on macOS,
  ~100 MB).
